# 實驗規劃 V2 — 歸零重設計(2026-07-08)

**原則聲明**:本規劃不引用任何過去實驗的結果作為證據或前提。所有需要的事實由本規劃內的實驗自行建立(多數只需數分鐘)。留用的只有工具鏈(Isaac Sim 6.0 環境、GMO 解析程式、runner 模式),因為工具不是證據。大方向不變:RTX Acoustic 超聲感測 × UR10e 機械手臂閉環 × 狀態估計,模擬可行性論文。

---

## 0. 設計哲學(為什麼這樣重排)

上一代程式的結構性錯誤:**先蓋控制、後驗感測**——控制迴路搭好了才發現感測器在那個幾何下讀不到目標。V2 反過來:

```
Stage 1  感測包絡量測    →  「感測器在什麼幾何條件下讀得到什麼」的地圖
Stage 2  任務設計進包絡  →  機器人任務擺在已驗證可感測的幾何裡
Stage 3  狀態估計        →  兩軌:目標狀態(包絡內)+ 自體狀態(誠實定位)
Stage 4  論文組裝        →  特性化 + 包絡 + 閉環 + 邊界(負結果)四章
```

**五條鐵律**(每個實驗 runner 的 header 必須逐條寫明如何滿足):

1. **配對對照**:任何「感測到 X」的宣稱,必須有 X-不存在(物理移除)的配對 run,同 session、同軌跡。
2. **資訊消融對照**:任何「閉環優於 Y」的宣稱,必須有致盲組(感測輸入置換為無資訊常數)跑相同管線。
3. **預註冊判準**:成功/失敗判準寫在 runner header,先於執行;主指標一律是「行為量 vs 目標量的相關/誤差」,不是 reason 標籤或到達率。
4. **原始波形落地**:每次擷取保存 primary-way 原始波形(npy),任何離線再分析不需重開模擬器。
5. **單一 claim 模式**:正式數據只有 acoustic_only(感測之外的任何 oracle 量僅得用於評估與記錄,不得進入控制、運動上限、或出口條件)。除錯用 scaffold 輸出必須寫進獨立目錄並在 summary 標記。

## Stage 1 — 感測包絡量測(論文第 4 章素材)

### S1:目標可偵測度地圖(核心實驗)

**問題**:在哪些幾何條件下,目標對 WPM 波形有超出雜訊的貢獻?

**方法**:因子化掃描,每個格點做配對擷取(有目標 / 物理移除目標),偵測度 = 配對差分能量 ÷ 同 session 雜訊底(由連續兩次同條件擷取定義)。

| 因子 | 水準 |
|------|------|
| 目標距離 | 0.15 / 0.3 / 0.5 / 0.8 / 1.2 m |
| 目標尺寸(立方) | 0.04 / 0.10 / 0.20 m |
| 感測器俯仰 | 水平 / 俯視 30° / 俯視 60° |
| 場景干擾 | 無 / 桌面 / 桌面+手臂 mesh(靜態擺位) |

- 全因子 5×3×3×3 = 135 格,先跑 **裂區子集 ~48 格**(距離全取,其餘半取),依結果加密。
- 單格一次 session 內完成配對(目標 RemovePrim / 重建),估 <1 分鐘/格。
- **產出**:偵測度熱圖 + 「可感測包絡」的邊界描述(例:俯視 X°、距離 ≤ Y m、目標 ≥ Z m、干擾條件 W 下 SNR > 10)。
- **預註冊判準**:SNR > 10 記為可偵測;整張圖無任何格點可偵測 → 大方向必須升級為「純特性化+負結果」論文(提前止損點 #1)。

### S2:包絡內的量測品質(感測器 datasheet)

只在 S1 判定可偵測的格點做:

- **距離編碼**:20 點距離掃描 × 3 重複 → peak_idx-距離線性度(r、RMSE、bias),重建取樣週期(自行校正,不引用舊值)。
- **側向編碼**:目標橫移 ±0.15 m 掃描 → 雙 RX 能量差是否單調編碼側向偏移(閉環側向對準的物理基礎)。
- **重複性**:同條件 10 次,特徵變異係數。
- **預註冊判準**:距離 r ≥ 0.95 且側向單調 → Stage 2 全開;僅距離達標 → Stage 2 做 1-DOF 版。

### S3:遮蔽機制歸因(論文「邊界」章素材)

S1 中「桌面+手臂」格點若不可偵測,逐因子拆解:只加桌 / 只加手臂 / 手臂不同關節姿態 / 感測器前移離開夾爪陰影。**產出**:不可偵測的機制解釋(指向、遮蔽、多徑淹沒各占多少),此為負結果章的核心數據。

**Stage 1 總成本估計**:~80–120 個短 session,GPU 時間 2–4 小時,可一夜批次。

## Stage 2 — 閉環任務(設計進包絡)(論文第 5 章素材)

**前提**:場景與感測器擺位由 S1/S2 的包絡決定,不繼承任何舊場景。候選(依 S1 結果三選一,決策記入 decisions):
(a) 感測器俯視構型,接近末段垂直下降時做距離伺服;
(b) 目標墊高脫離桌面平面(pedestal),水平接近;
(c) 合作目標(較大反射體,對應實務中的超聲信標)。

### D1:1-DOF 閉環接近(主實驗)

- 目標位置隨機化(範圍 = 包絡內),n = 30。
- **三臂設計**:閉環(聲學)/ 致盲(同管線、感測置換常數)/ 開環(預設固定行程)。
- **主指標(預註冊)**:r(停止位置, 目標位置) ≥ 0.9 且顯著高於致盲組;RMSE(停止距離 − 目標 standoff) 報告 CI。
- 到達率只作次指標。致盲組與閉環組主指標無差 → 判定失敗,回 Stage 1 檢討(提前止損點 #2)。

### D2:2-DOF(前向 + 側向伺服)

僅當 S2 側向編碼達標。同三臂設計,加側向停止誤差指標。

### D3:端到端夾取

D1/D2 通過後,接 Robotiq 夾取 + 升舉,n = 30。**報告分解**:P(接近對位成功)× P(夾取|對位)——兩個條件機率分開宣稱,不給單一混合成功率。

**Stage 2 總成本**:每臂 30 episodes、每 episode 秒級到半分鐘,單實驗 <1 小時。

## Stage 3 — 狀態估計(論文第 5–6 章素材)

### P1:聲學自體狀態估計(誠實新定位)

聲學特徵 → 感測器/手臂位姿回歸(離線,新采數據或 Stage 2 過程數據)。宣稱上限:「聲學特徵編碼機器人自體構型」。

### P2:目標狀態估計(僅包絡內)

- 資料集必含 **無目標 negative trials**(上一代缺的關鍵對照)。
- 第一道預註冊關卡:目標存在偵測 AUC;≈0.5 → 停止,不做後續距離/區域分類宣稱。
- 通過後才做 near/far、stop-region 分類,特徵消融(聲學 / 位姿 / 全部)照舊法但加 negative 對照。
