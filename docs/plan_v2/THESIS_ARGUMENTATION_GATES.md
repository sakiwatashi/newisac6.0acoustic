# 論文論證閘門（Argumentation Gates）

> **用途**：補上「合規／數字／禁詞已過，但論證仍可被整段重寫」的缺口。  
> **何時用**：任一節大修、整章貼回正式 docx、口試前全文過一遍、外部審稿回饋後。  
> **與既有閘門的關係**：  
> - 實驗鐵律 → `docs/WPM_EXPERIMENT_RULES.md`、預註冊 runner header  
> - 數字／宣稱／禁詞 → 既有 rebuild 驗收  
> - **本文件只管「論證職能、讀者路徑、證據閉環」**  
> 三層全綠才算「論述可對外」。

**版本**: 1.0 | **日期**: 2026-07-22

---

## 0. 使用方式（強制流程）

每節（或每次改動）依序：

1. **G0 定位**：寫下本節一句職能（見下）。寫不出來 → 不准開寫。  
2. **G1–G5 自檢**：逐條勾選；失敗項必須「刪／移／改寫」，不可只加說明句糊弄。  
3. **G6 全章線**：該章所有節 G0 串起來必須成一條論證鏈。  
4. **記錄**：在 PR／交接／commit message 寫「已過閘門：章節 x.x」；未過不得標「定稿」。

**禁止**：用「字數夠了／引用齊了／術語有解釋」取代本閘門。

## G0 — 節職能（One Job）

| 欄位 | 必填 |
|------|------|
| 節號／標題 | |
| **本節只解決的一個問題**（≤30 字） | |
| 讀者讀完應能用一句話回答 | |
| 本節**明確不負責**的事（至少 2 項） | |
| 若刪半，保留哪一段仍成立？ | |

**不過關條件**：

- 本節同時想做「背景 + 方法定義 + 結果預告 + 未來工作」  
- 「不負責」寫不出來（代表什麼都想塞）

## G1 — 論證鏈（Why → Gap → What we do）

本節（尤其 1.1、2.8、6.x）必須能標出句子角色：

```text
應用／現象 → 困難／混淆 → 既有不足（gap）→ 本文作法 → 驗證方式
```

**檢查**：

- [ ] 每一段可標唯一角色：`背景` / `困難` / `缺口` / `作法` / `驗證` / `邊界`  
- [ ] **缺口**至少有一句獨立成段，不能只暗示  
- [ ] 作法句可對到後文某一章／某一實驗，不懸空  
- [ ] 沒有「跳級」：未定義問題就進入實作名詞堆

**不過關**：讀完 1.1 仍無法回答「若不做本研究，學術／方法上缺什麼」。

---

## G2 — 讀者路徑與減肥（Cut Test）

**檢查**：

- [ ] 連續兩段不重複同一主張（只換詞重說 = 刪一段）  
- [ ] 任一專有名詞首次出現時：讀者**已經**知道為何需要它  
- [ ] **可移測**：該細節移到更後章節後，本節論證仍完整 → 必須移走  
- [ ] 刪除「宏觀趨勢段」（智慧製造／數位孿生／成本）後，問題意識是否更清楚？若是 → 刪或縮成半句  
- [ ] 無「展示型」清單：列了平台／模組名稱但後面不用來推進論點

**強制減肥題**（改完必須能回答）：

> 刪掉哪三段，本節職能（G0）仍然成立？

答不出三處可刪 → 多半仍在堆資訊。

## G3 — 主張–證據閉環（Claim–Evidence）

對本節每一個**可檢驗主張**（含「可／不可」）：

| 主張（原文摘一句） | 證據位置（章節／報告／數據路徑） | 證據類型* | 狀態 |
|--------------------|----------------------------------|-----------|------|
| | | 實驗／文獻／定義／推論 | 閉環 / 缺口 |

\*類型：

- **實驗**：有數據或裁定報告  
- **文獻**：引用支撐的是「文獻結論」而非「我們的推論」  
- **定義**：操作定義，不需外部數據  
- **推論**：必須標成推論，不可寫成事實  

**檢查**：

- [ ] 無「裸主張」：正文說「已確認／證明／顯示」卻無指向  
- [ ] 文獻引用的後半句若是本文推論，改為「據此，本文認為…」  
- [ ] repo 有、正文無 → 算**未閉環**（外審只讀論文）  
- [ ] 絕對用語（不受影響、完全、首次、等價）有實驗或文獻直接支撐，否則降級

**P0 特則（3.1／方法）**：  
「GMO → 序列 → 峰 → 距離」鏈的四步，每步必須在正文有：官方／實測語意 + 本文操作 + 證據指標。缺一步 = 3.1 不過閘。

---

## G4 — 尺度一致（Scope Match）

**檢查**：

- [ ] 開頭問題尺度 ≤ 實驗實際尺度（近距走廊接近 ≠ 工廠數位孿生部署）  
- [ ] 未驗證的架構（VLM 階層、實機 CH201、摩擦夾持）不進「本研究完成」語氣；最多一句定位或未來工作  
- [ ] 摘要／1.1／貢獻條／6.2 宣稱集合一致（改一處必掃三處）  
- [ ] 比喻性用語（最後一公尺等）若與實驗區間不符 → 改為操作定義

---

## G5 — 句子服務論點（Micro, but tied to G0）

不只「通順」，而是**每句是否服務 G0**：

- [ ] 段首句 = 段旨；段尾句 = 接到下一功能（困難→缺口或作法→驗證）  
- [ ] 術語解釋緊貼「為何本節需要它」，禁止為解釋而解釋的百科段  
- [ ] 中英／超音波用語全節一致  
- [ ] 公式首次出現：符號定義 + 適用假設（單站／雙站／經驗映射）同段寫清  

**可讀性舊閘門仍保留**（術語首現、禁詞），但 **不能單獨當作本 G5 通過**。

## G6 — 章級串線（Chapter Thread）

把該章每節 G0 排成表：

| 節 | 一句職能 |
|----|----------|
| | |

**檢查**：

- [ ] 鄰節職能不重疊超過 30%  
- [ ] 前節輸出 = 後節輸入（例如 1.1 缺口 → 1.2 RQ；3.1 鏈結論 → 3.2 特徵）  
- [ ] 無「結果章重講方法章 80%」  
- [ ] 負結果／限制出現在該出現的章，不與正面結果搶同一職能段

## 全稿快速掃描（口試前 30 分鐘版）

1. 只讀摘要 + 1.1 + 1.4 貢獻 + 6.2 邊界：能否講完「問題—作法—證據—不宣稱」？  
2. 只讀 3.1：能否講完「讀到的是什麼、為何能當距離」？  
3. 只讀 5.x 表與消融：能否講完「為什麼不是捷徑成功」？  
4. 任抽 5 句「顯示／證明／確認」：能否 30 秒指到數據？  

任一失敗 → 回對應 G1–G3，**不要**先美化用詞。

## 與實驗／P0 的交接

| 層級 | 文件 | 何時擋下「定稿」 |
|------|------|------------------|
| 感測鏈 P0 | `docs/plan_v2/reports/P0_GMO_CHAIN_AUDIT.md` | 3.1 宣稱超過 P0 已閉環部分 |
| 實驗正典 | `STATUS.md`、`docs/plan_v2/reports/*` | 結果章數字與正典目錄不一致 |
| 論證 | **本文件** | G0–G6 未勾選 |

**原則**：P0 或實驗未過 → 禁止用「寫得更順」當定稿；論證未過 → 禁止用「數字全綠」當定稿。
