## 側向編碼:證偽(誠實負結果)

way-序數能量差(bal)在 ±0.15 m 橫移下無單調結構(−6.0 至 −3.7 亂序,ρ=0.357)。結合實測「GMO 所有 way 標籤 (tx,rx,ch)=(0,0,0)」:**此 rxGroup 配置下雙 RX 側向編碼不可用** → **D2(2-DOF 側向伺服)剪除**,D1 做 1-DOF。未來工作:雙 rxGroup(g001:[0]、g002:[1])配置或許可分離接收器,未驗證。

## 重要校正發現:樣本週期 ≈ 103 µs,不是舊說的 132.5 µs

V2 自校(兩種高度獨立測得 103.1/100.8 µs)接近 schema 預設 102.4 µs,**與舊規則文本(`docs/WPM_EXPERIMENT_RULES.md` 規則 1-4)的 132.5 µs 不符**。V2 原則「不沿用舊校正」在此直接兌現價值——舊值若沿用,所有距離估計會系統性偏大 29%。舊 132.5 的來源(7/6 armfree,不同場景/參數)與本測的差異未歸因,列為開放問題;**V2 後續一律使用本輪自校值(隨 datasheet_summary.json 落盤)**。

## 量測方法學補充(接手必讀)

1. GMO 每幀含 2 個 way,**id 欄全零**(接收器身分不編碼於 id);本套件以 way 序數拆分,該假設已被側向掃描證偽為「不含空間資訊」——但 primary(取 peak 較大者)不受影響。
2. 幀間輪替結構 → 量測 block 要 ≥12 幀(6 幀時 drift 8%,12 幀 3.9%)。
3. 首版 runner 因「同幀需含雙 RX」假設全 NaN——教訓:對幀內結構的任何假設先用 S2_DEBUG_IDS=1 實測。
